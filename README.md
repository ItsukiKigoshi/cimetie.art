## 対象

- 美術館  
   - 地方美術館を盛り上げられたら最高だね
- ギャラリー？？  
   - 購入した商品をどこに展示しているか使用者が申請できるなどしてアート市場で作品の価値を高める工夫も出来そう  
   - 小規模なアーティストも支援したい
- （博物館?）: 作品単位で扱うなら博物館は含まない方が一貫性ある
- 芸術祭も
- 行きたいのは美術館自体の時もあるしある展示だったりもするのでどちらも取り込みたい
- このサービスの対象として演劇も隣接領域ではあるがそこまで含めるか？？
- 作品リスト?  
   - "この作品を実際に見たい"という需要に対しては有効か?, 作品がどこで転じされてきたかとこれからどこで展示されるか見られる  
   - あまりにも数が膨大なので, どこまでカバーするかは要検討  
   - 最初の段階で要らないだろう  
   - 展覧会の作品リストから所蔵元が公開しているデータベースにリンク (サイトで画像は保持しない)

## 設計

- まずはHTML+JS+RESTでMVP
- 展覧会単位でミュージアムの訪問歴を記録できる: 世界対応 , ログイン，美しいデザイン，個人開発ではなく法人で  
   - ページ: 各美術館, 各展覧会のページ  
   - 行きたい, 行った,
- OSMから地球儀?: radio garden; 自己満. ユーザ体験が下がれば元も子もない.  
   - [earth.cimetie.art](http://earth.cimetie.art)という裏ページにしてどこか隠しボタンみたいなところから見える, とかがちょうどいいかも.
- 訪問日+美術館名を入れれば会期中の候補の展覧会が出てくる
- ソーシャル機能よりも，データベースとして機能するものを. ソーシャルではなく個人の体験で完結しつつ，他人も覗ける, くらいの距離感.  
   - レビューサイトではない
- ブクログなど日本のサービスにありがちなごちゃごちゃを廃し，AI味ではなく人の美しさを
- (批評家の)批評へもアクセスできる? 美術手帳, artforum, tokyoartbeat?
- Google Arts&Culture, 美術手帳, 各美術館のDB, ジャパンサーチ, artsy, [artplatform.go.jp](http://artplatform.go.jp)?と連携
